Design Section 2 — /gh Coordinator

Entry point, routing logic, and chained workflow support

Invocation examples
/gh repo new my-project
/gh repo new my-project private=true
/gh commit "add hero section"
/gh commit revert a3f2c1
/gh commit --all
/gh pr open
/gh pr open issue=42
/gh pr merge
/gh pr merge auto=true
/gh pr status
/gh                   # no args → interactive menu
Routing logic
/gh repo …
→ invokes gh-repo with remaining args
/gh commit …
→ invokes gh-commit with remaining args
/gh pr …
→ invokes gh-pr with remaining args
/gh (no args)
→ interactive menu — choose action
Chained workflows (coordinator only)
/gh init      → repo new → initial commit → done
/gh ship      → commit → push → pr open → (auto-merge if approved)
/gh ship issue=42 → gh-issue fetch → commit → push → pr open (linked to #42)
Chained commands are the coordinator's superpower — specialists handle their phase, coordinator sequences them without stopping.
