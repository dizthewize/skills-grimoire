references/ — what each file contains
conventions.md
The master human-readable reference. Branch naming rules, commit format, PR standards, merge strategy, label taxonomy. This is what a new developer (or AI) reads to understand how this project manages GitHub.
commit-template.md
Standard types (feat/fix/chore/docs/refactor/perf/test/style) + custom additions (build/deploy/skill). Scope rules, description length, breaking change footer format. Read by gh-commit for validation.
pr-template.md
PR body sections (Summary, Changes, Closes, Checklist), label-to-commit-type mapping, reviewer assignment rules, merge strategy (squash default). Read by gh-pr when generating PR bodies.
repo-defaults.md
Branch protection ruleset, default labels + hex colors, README boilerplate sections, .gitignore defaults. Read by gh-repo at creation time. Projects can override per-repo.
Using the suite in another project
# Install via Claude Code plugin system
claude plugin add github.com/[you]/gh-management
# Or clone references into project for customization
cp gh-management/references/ ./references/
# Then just use it
/gh repo new my-new-project
/gh commit "feat: add homepage"
/gh ship issue=12
