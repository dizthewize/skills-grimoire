Design Section 4 — site-builder

Parallel research agents first, then phased implementation with per-phase commits.

Phase R — parallel research agents (all run simultaneously)
Stack Scout
Confirms framework setup, dependencies, folder structure, config files needed
Component Analyst
Maps design-system.md specs to buildable components. Flags any gaps or ambiguities.
SEO Mapper
Maps site-plan.md keywords to page files. Plans meta tags, schema markup, sitemap.xml.
Copy Integrator
Structures copy from site-plan.md into component-ready text blocks, CTAs, and form labels.
Auth + DB Planner
Only active if auth/Supabase selected. Plans schema, RLS policies, env vars, auth flow pages.
Conversion Optimizer
Plans CTA placements, sticky elements, click-to-call, exit-intent, form positions across pages.
All agents report back → coordinator synthesizes into build-plan.md → user sees summary → build begins
Phase B — sequential build (each phase = 1 commit)
B1
Scaffold — init framework, install deps, configure Tailwind/CSS vars from design-system.md, folder structure, env file template
B2
Shared components — header (sticky, logo, phone, CTA), footer (contact, service area, socials), mobile sticky CTA bar, layout wrapper
B3
Homepage — hero, lead capture form, offer section, services grid, trust signals, testimonials, service area, final CTA. All copy wired from site-plan.
B4
Inner pages — Services, About, Contact, Service Area pages per site-plan.md hierarchy
B5
SEO layer — meta tags, schema markup (LocalBusiness JSON-LD), sitemap.xml, robots.txt, OG tags
B6*
Auth + Supabase — only if selected. Sign up/in pages, RLS policies, env wiring, protected routes
B7
Deploy prep — netlify.toml or vercel.json, redirect rules, form handling config, build test
State saved to .web-studio/state.json after each phase — resumable if interrupted
B6 is skipped entirely when no auth/database selected
