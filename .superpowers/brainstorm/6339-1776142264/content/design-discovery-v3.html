Design Section 2 — site-discovery (v3, streamlined)

User answers only what they uniquely know. Everything else is inferred or recommended by site-planner + site-designer.

Phase 1 — universal (all site types)
? What is the business name?
? What type of site is this?
→ Service-based business / SaaS / E-commerce / Blog / Content / Portfolio / Landing page
? What city or region is the business based in? (for local SEO + branding)
Phase 2A — service-based business
Only 5 questions — everything else is determined by site-planner & site-designer
? What trade or industry?
→ Flooring / Roofing / Remodeling / Landscaping / HVAC / Plumbing / Auto / Mobile Mechanic / Cleaning / Painting / Other (describe)
? What is the primary contact phone number?
Critical — appears in header, hero, footer, and sticky bar on every service site.
? What cities or areas do you serve?
Used for service area section + local SEO keywords. Can be a list or radius description.
? Do you have brand colors? (or let the skill pick industry-appropriate colors)
? Any competitor or reference sites you admire? (optional — informs design direction)
Inferred by site-planner + site-designer (not asked)
✦ Services list — derived from industry
✦ Lead capture offer — recommended by site-planner based on trade norms
✦ Copy tone — recommended based on industry (e.g. bold/direct for auto, friendly/local for landscaping)
✦ Trust signals — standard for the trade (licensed & insured, years in business, etc.)
✦ Color palette — if no brand colors given, site-designer picks industry-appropriate scheme
✦ Page structure — site-planner recommends based on site type
✦ Urgency/scarcity copy — site-planner writes based on industry conventions
Phase 2B — SaaS branch
Target persona, pricing model, key differentiators, trial/freemium, integrations, stack…
Phase 2C — E-commerce branch
Product catalog, payment processor, shipping, collections structure…
Phase 3 — tech decisions (all types)
? Frontend stack?
Skill recommends based on site type (Astro → service/blog, Next.js → SaaS/e-comm). User can override.
? Does this site need user accounts / authentication?
→ No (most service sites) / Yes — Supabase Auth / Yes — other
? Does it need a database or backend?
→ No / Yes — Supabase (credentials requested now) / Yes — other
? Deployment platform?
→ Netlify via GitHub (default) / Vercel (credentials requested)
Output
→ writes docs/product-marketing-context.md
→ summarizes captured info + what site-planner will infer/recommend
→ waits for user confirmation before proceeding
