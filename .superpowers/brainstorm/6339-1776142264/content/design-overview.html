Design Section 1 — Overall Flow

How a single /web-studio invocation produces a live website

┌─────────────────────────────────────────────────────────────┐
│ User runs: /web-studio (or passes GitHub issue URL) │
└──────────────────────────┬──────────────────────────────────┘
│
▼
web-studio (coordinator)
reads GitHub issue → validates prerequisites → invokes phases in order
① site-discovery
Intake Q&A
→ product-marketing-context.md
→ GitHub milestone created
② site-planner
→ site-architecture
→ seo-audit
→ copywriting
③ site-designer
→ ui-ux-pro-max
→ frontend-design
④ site-builder
5-6 parallel research agents → phased build → per-page commits
Supabase schema + auth wired during build
⑤ site-deploy
Netlify (default)
or Vercel w/ credentials
→ GitHub PR → auto-deploy
Shared: ./product-marketing-context.md written by site-discovery, read by all downstream skills
State: .web-studio/state.json — tracks phase completion, enables resume
GitHub: issue read at start → sub-issues + milestone updated after each phase
