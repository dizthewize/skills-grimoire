Design Section 2 — site-discovery intake questions

Everything asked before any planning or building begins. Output: docs/product-marketing-context.md

Phase 0 — asked by web-studio at invocation
? Deploy when done? (yes / no / ask-me-later)
Phase 1 — site-discovery: business context
? What is the name of the business or product?
? What type of site is this?
→ SaaS / E-commerce / Service-based / Blog / Portfolio / Landing page / Other
? Who is the target audience? (brief description)
? What is the primary goal of the site? (convert leads / sell products / educate / etc.)
? Describe the business in 2–3 sentences.
Phase 2 — site-discovery: tech decisions
? Which frontend stack?
→ Next.js (recommended for SaaS/e-comm) / Astro (recommended for blogs/marketing) / Vite+React / Other
? Does the site need user authentication?
→ No (public site) / Yes — Supabase Auth / Yes — other (describe)
? Does the site need a database or backend?
→ No / Yes — Supabase (preferred) / Yes — other (describe)
↳ If Supabase: request SUPABASE_URL + SUPABASE_ANON_KEY now
? Preferred deployment platform?
→ Netlify (default, via GitHub integration) / Vercel (request credentials)
Phase 3 — site-discovery: content & design hints
? Do you have brand colors, fonts, or a style reference? (optional)
? List the pages you know you need. (or leave blank — site-planner will suggest)
? Any competitors or sites you admire? (optional — informs design direction)
? What tone should the copy have?
→ Professional / Friendly & casual / Bold & direct / Luxurious / Playful / Technical
Output
→ writes docs/product-marketing-context.md
→ writes .web-studio/state.json (phase: discovery complete)
→ summarizes back to user for confirmation before proceeding
