Design Section 5 — site-reviewer + site-deploy (revised)

A team-based PR review gate between build and deploy. Only surviving findings ship.

Updated flow — build → review → deploy
④ site-builder   
↓ pushes branch + opens GitHub PR
⑤ site-reviewer   ← NEW
↓ PR approved with surviving findings only
⑥ site-deploy    (optional)
⑤ site-reviewer
TeamCreate + SendMessage — adversarial review loop
Frontend Specialist
agent: code-refactorer
Reviews: component structure, CSS vars, responsive layout, Tailwind usage, accessibility
SEO + Performance Reviewer
agent: general-purpose
Reviews: meta tags, schema markup, Core Web Vitals, image optimization, sitemap
Conversion + Copy Auditor
agent: general-purpose
Reviews: CTA placements, click-to-call, form UX, copy accuracy vs site-plan.md
Security + Deploy Reviewer
agent: general-purpose
Reviews: env var handling, form security, Supabase RLS (if used), deploy config
Devil's Advocate  agent: code-refactorer
Receives all 4 reviewer reports → challenges every finding with:
  · "Is this actually a problem in context?"
  · "Does this match the site-plan.md spec or contradict it?"
  · "Is the severity justified given the site type?"
  · "Could this be a false positive from the design-system.md choices?"
Defense round — original reviewers respond
Each finding → DEFENDED (stays, confidence rated) | ADJUSTED (revised) | WITHDRAWN
Final report — PR comment posted to GitHub
✦ Only surviving findings listed, each with confidence: HIGH / MEDIUM / LOW
✦ Withdrawn + overruled findings logged separately (not blocking)
✦ Critical findings → auto-fix attempted by coordinator before PR approval
✦ Medium/low findings → listed as suggestions, PR approved
✦ PR merged → site-deploy proceeds (if enabled)
⑥ site-deploy
optional — Netlify (default) or Vercel
Triggers after PR merge → polls for live URL → posts deploy summary
Returns: live URL, page count, SEO checklist, recommended next steps
Future companion — /web-issue
GitHub Projects-aware feature + bug skill. Reads issues/milestones, invokes specialist skills, writes progress back. Built after /web-studio is in real use.
